sed 原理 
【原理】sed 处理文件的时候
  • 每一行都被保存在一个叫模式空间的临时缓冲区中。
  • 除非行被删除或者输出被取消，否则所有被处理的行都将 打印在屏幕上。
  • 接着模式空间被清空，并存入新的一行等待处理。
  • h 命令取出模式空间的行，然后复制并存入一个称为保持空间的特殊缓冲区内。
  • H 命令取出模式空间的行，然后复制并追加到被称为保持空间的特殊缓冲区内。
  • g 命令取出保持空间的行，然后替换掉当前存在于模式空间中的行。
  • G 命令取出保持空间的行，然后追加到当前存在于模式空间中的行的末尾。
【例子】任何包含test的行都被复制并追加到该文件的末尾
sed -e '/test/h' -e '$G' file
  • x 命令互换模式空间和保持空间的内容
【例子】把包含test与check的行互换
sed -e '/test/h' -e '/check/x' file
